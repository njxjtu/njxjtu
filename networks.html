  • Cisco IOS Router OSPF Neighbor L3 Link

    Cisco OSPF config pre-condition:
    Make sure :
    1. There is physical link between serial2/3 on 2.135 and serial2/3 on 2.138
    2. devices could ping each other(on 2/135, could ping 120.20.20.138; on 2.138, could ping 120.20.20.135)
    3. interfaces(serial2/3) on both devices are not shutdown
    Check the OSPF config on Cisco device:
    Show run | s ospf
    OSPF config on 192.168.1.138:
    OSPF config on 10.128.2.135(admin/D0rado!!/enable/enable):
    Confirm / check ospf neighbor on Cisco:
    Show ip ospf neighbor
    Example for Cisco OSPF: 10.128.2.30-10.128.2.31
    On 10.128.2.30 (management interface: vlan1)
    Router ospf 1
    Network 10.128.2.30 0.0.0.255 area 0
    On 10.128.2.31 (management interface vlan1)
    Router ospf 1
    Network 10.128.2.31 0.0.0.255 area 0
    %%since x.x.x.30 and x.x.x.31 has 30 and 31 as the difference; therefore the mask would be 0.0.0.255
